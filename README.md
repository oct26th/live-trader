# Live Trader — BTC Regime-Aware Crypto Trading Bot

## 設計構想與原理

### 核心問題

比特幣市場同時存在兩種截然不同的環境：

| 環境 | 市場特徵 | 策略表現 |
|------|----------|----------|
| **趨勢市場** (Trending) | ADX > 40，單向波動 | 追蹤趨勢策略表現優異 |
| **區間市場** (Ranging) | ADX < 25，反覆震盪 | 均值回歸策略更有效 |

大多數交易 Bot 採用單一策略參數，忽略市場環境切換，導致在單一環境中表現良好的策略在環境轉換時虧損。

### 設計核心：雙軌進場 + BTC 宏觀環境切換

```
                    ┌─────────────────┐
                    │  BTC/USDT 4H    │
                    │  計算 ADX 判斷   │
                    │  市場環境        │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              ▼              ▼              ▼
        ADX < 25        25 ≤ ADX ≤ 40    ADX > 40
        (區間)          (過渡)           (趨勢)
         ↓                   ↓                ↓
    ───────────────────────────────────────────
    │  均值回歸軌 (PULLBACK)  │  趨勢追蹤軌 (TREND) │
    │  RSI < 38              │  MA 黃金交叉        │
    │  MA 偏多               │  MACD > 0          │
    │  MACD > 0              │  RSI 40-70         │
    │                        │  ADX > 25          │
    ───────────────────────────────────────────
              │                        │
              └───────────┬────────────┘
                          ▼
                   進場信號評估
                   (費用過濾器)
```

### 雙軌進場邏輯

**軌跡一：均值回歸 (Pullback)**
- 適用場景：市場回調時進場
- 進場條件：RSI < 38 且 MA 偏多 且 MACD > 0
- 邏輯：超賣後反彈，逆勢短線

**軌跡二：趨勢追蹤 (Trend)**
- 適用場景：明確趨勢形成後進場
- 進場條件：MA 黃金交叉 且 MACD > 0 且 RSI 40-70 且 ADX > 25
- 邏輯：順勢而為，趨勢延續時持有

### BTC 環境切換參數組

| 參數 | 區間 (ADX<25) | 過渡 (25-40) | 趨勢 (ADX>40) |
|------|---------------|--------------|---------------|
| 允許趨勢軌 | ❌ | ✅ | ✅ |
| 最大持倉數 | 2 | 1 | 3 |
| 止損 | -3% | -4% | -5% |
| TP1 | 5% | 6% | 8% |
| TP2 | 10% | 12% | 15% |

### 費用過濾器

進場前檢查：TP2 淨收益（扣除來回 0.6% 交易所費用）必須超過 3%。

```
TP2_net = TP2 - 2 × FEE_RATE = 0.10 - 0.012 = 0.088 (8.8%)
門檻：MIN_NET_TP2 = 0.03 (3%)
結果：8.8% > 3% → 允許進場
```

### 為何用 BTC 判斷宏觀環境？

加密貨幣市場 90% 跟隨比特幣走勢。監控 20 個幣種的市場結構是冗餘的——它們最終都反映 BTC 的方向。專注監控 BTC/USDT 的 ADX，大幅降低 API 請求次數，同時抓住核心市場脈動。

### 歷史回測驗證

| 策略版本 | 180 天報酬 | 最大回撤 | 交易次數 | 勝率 |
|----------|------------|----------|----------|------|
| ORIGINAL | -8.8% | -27.1% | 204 | 14% |
| **Variant C** | **-0.4%** | **-13.3%** | **27** | **25%** |

Variant C 移除 1H 趨勢信號（主要虧損來源），保留 4H 結構，勝率提升至 25%，回撤減少一半。

### 風險控制

1. **動態止損** — 根據市場環境調整止損幅度
2. **分階段獲利了結** — TP1 先鎖定部分利潤，TP2 持有至趨勢反轉
3. **最大持倉限制** — 最多 3 個活躍倉位，保留 20% 現金
4. **費用過濾** — 避免低波動低利潤交易
5. **14 天輪換** — 避免長期持有暴露於黑天鵝事件

### 技術棧

- **語言**: Python 3
- **交易所**: Binance (現貨)
- **數據**: CCXT 獲取 K 線數據
- **調度**: Cron (每小時運行一次)
- **通知**: Discord Webhook (每小時報告)

### 代碼結構

```
live_trader_v3.py
├── _fetch_ohlcv()          # 獲取 K 線數據
├── _btc_regime()           # 計算 BTC ADX 判斷環境
├── _apply_regime()         # 根據環境選擇參數組
├── _check_entry_pullback() # 均值回歸進場信號
├── _check_entry_trend()    # 趨勢追蹤進場信號
├── _check_stop_loss()      # 止損檢查
├── _check_take_profit()    # 分階段止盈檢查
└── run_cycle()             # 主循環：獲取數據 → 計算信號 → 執行訂單
```
